Foundations

How JS runs in the browser

Python needs an interpreter — you run python script.py and the interpreter executes it. JavaScript is built into every browser. You write code inside an HTML file (or link to a .js file), open the file, and it runs. No install, no terminal. Open DevTools with F12 → Console to see output and try one-liners interactively — it's like Python's REPL.

In the Dandelions project, index.html has <script src="game.js">. The browser loads game.js, runs it top-to-bottom, and everything defined at the top level is live in the page's global scope.

const, let, and var

JS has three ways to declare a variable. You should use two of them:

  • const — the binding can't be reassigned. Use this by default.
  • let — the binding can be reassigned. Use when you need to update the variable.
  • var — old, function-scoped, hoisted in ways that cause subtle bugs. Never use it in new code.

Neither const nor let has a direct Python equivalent — Python just has assignment and no enforcement. Think of const as a hint to yourself and to anyone reading the code that this value won't change.

JavaScript
const SIZE = 5;
let round = 0;
round = 1;      // ✓ let can reassign
SIZE = 6;       // ✗ TypeError: const
Python
SIZE = 5        # convention only
round = 0
round = 1       # fine
SIZE = 6        # also fine, no enforcement

2D arrays: JS vs numpy

Python game code might use numpy for grids, but JS has no numpy. JS arrays are regular lists — and a "2D array" is just an array of arrays.

JavaScript
const board = [
  [0, 0, 0],
  [0, 1, 0],
  [0, 0, 0]
];
board[1][1];  // → 1
board[1][2] = 2;
Python (plain list)
board = [
  [0, 0, 0],
  [0, 1, 0],
  [0, 0, 0]
]
board[1][1]   # → 1
board[1][2] = 2

The indexing is identical. The big difference shows up in creation and copying — you can't use numpy's np.zeros((5,5)) or board.copy(). You'll see how the engine handles this in createBoard and copyBoard.

Truthiness and falsiness

Both languages have truthy/falsy values, but the falsy sets differ. In JS, falsy values are: false, 0, "", null, undefined, and NaN. Notice 0 is falsy — which matters a lot in board logic since empty cells are 0.

Important: Use === (strict equality) in JS, not ==. The loose == does type coercion: 0 == false is true, which causes bugs. 0 === false is false. Stick to === everywhere — it behaves like Python's ==.

The DIRECTIONS object — JS dict

The engine's constant DIRECTIONS is a plain JS object used as a dictionary. Its keys are strings and its values are [dr, dc] arrays.

JavaScript
const DIRECTIONS = {
  N:  [-1,  0],
  NE: [-1,  1],
  E:  [ 0,  1],
  // …
};
DIRECTIONS['N'];   // [-1, 0]
DIRECTIONS.N;      // same thing
Object.keys(DIRECTIONS); // ['N','NE','E',…]
Python
DIRECTIONS = {
  'N':  (-1,  0),
  'NE': (-1,  1),
  'E':  ( 0,  1),
  # …
}
DIRECTIONS['N']    # (-1, 0)
# no dot access for dicts
list(DIRECTIONS.keys())  # ['N','NE','E',…]

JS objects can be accessed with dot notation (obj.key) or bracket notation (obj['key']). Use bracket notation when the key is in a variable: DIRECTIONS[dir] where dir = 'NE'.

1 createBoard




  Array.from()
  arrow functions
  Array.fill()




function createBoard() {
  return Array.from({ length: 5 }, () => Array(5).fill(0));
}



What it does



  Creates and returns a fresh 5×5 board — a 2D array where every cell is 0 (empty).
  No parameters, nothing to configure, just a clean slate.




The JS concepts



  Array.from({ length: 5 }, callback) is the JS idiom for "make an array by calling
  a function N times." The first argument is any object with a length property;
  the second argument is a mapping function called once per index.




  () => Array(5).fill(0) is an arrow function. It takes no
  arguments (the ()) and returns a 5-element array filled with zeros.
  In Python you'd write lambda: [0]*5, though the more idiomatic Python
  would be a list comprehension: [0]*5.





  Why not just Array(5).fill(Array(5).fill(0))?

  Because fill with an object argument fills every slot with the same
  reference. Modifying one row would modify all rows. The arrow function creates a fresh
  inner array each time — the equivalent of the Python gotcha with
  [[0]*5]*5 vs [[0]*5 for _ in range(5)].





  

    
JavaScript

    
Array.from({ length: 5 },
  () => Array(5).fill(0)
);

  

  

    
Python

    
[[0] * 5 for _ in range(5)]

# NOT [[0]*5]*5 — same bug applies

2 copyBoard




  Array.map()
  spread operator
  shallow vs deep copy




function copyBoard(board) {
  return board.map(row => [...row]);
}



What it does



  Returns a deep copy of the board. Changes to the copy won't affect the original,
  which is critical because blowWind and placeFlower
  both need to return a new board without mutating the input.




The JS concepts



  array.map(fn) applies a function to every element and returns a new array.
  In Python this is either [fn(x) for x in array] or map(fn, array)
  (though Python's map returns an iterator, not a list). JS's .map()
  always returns a new array immediately.




  [...row] is the spread operator inside an array literal.
  It expands an iterable into individual elements, creating a shallow copy of the row.
  Since rows contain numbers (primitives), a shallow copy is a true copy here.
  In Python the equivalent is row[:] or list(row).





  

    
JavaScript

    
board.map(row => [...row])

// spread also works for merging:
const a = [1, 2];
const b = [3, 4];
[...a, ...b]; // [1,2,3,4]

  

  

    
Python

    
[row[:] for row in board]

# unpacking equivalent:
a = [1, 2]
b = [3, 4]
[*a, *b]   # [1,2,3,4]

  






  Why not JSON.parse(JSON.stringify(board))? That's a common
  deep-copy hack in JS that works but is slow and breaks for non-JSON-safe values.
  For a 2D array of numbers, map + spread is clean and fast.

3 placeFlower




  === strict equality
  returning null
  immutable pattern




function placeFlower(board, row, col) {
  if (board[row][col] === 1) return null;
  const newBoard = copyBoard(board);
  newBoard[row][col] = 1;
  return newBoard;
}



What it does



  Attempts to place a flower at (row, col). Flowers can go on empty cells
  (0) or seed cells (2), but not on an existing flower (1).
  Returns the new board on success, null on failure.




The JS concepts



  === is strict equality — it checks value and type with no coercion.
  Always use this instead of ==. Here board[row][col] === 1 is
  exactly like Python's board[row][col] == 1.




  null in JS is a deliberate "no value" signal, similar to Python's None.
  Returning null on an invalid move is a common pattern — the caller checks
  if (result !== null) or just if (result).




  Notice the function never mutates the input board. It calls copyBoard first,
  then modifies the copy. This is the immutable data pattern — functions that return new
  state instead of changing existing state. It makes the game easy to reason about since
  you can always keep a "before" snapshot.





  

    
JavaScript

    
if (board[row][col] === 1) return null;
// caller:
const result = placeFlower(board, 2, 2);
if (result) { /* success */ }

  

  

    
Python

    
if board[row][col] == 1: return None
# caller:
result = place_flower(board, 2, 2)
if result is not None: ...

4 blowWind




  destructuring assignment
  while loop
  nested for loops
  let for mutation




function blowWind(board, direction) {
  const [dr, dc] = DIRECTIONS[direction];
  const newBoard = copyBoard(board);
  for (let r = 0; r < 5; r++) {
    for (let c = 0; c < 5; c++) {
      if (board[r][c] === 1) {
        let nr = r + dr;
        let nc = c + dc;
        while (nr >= 0 && nr < 5 && nc >= 0 && nc < 5) {
          if (newBoard[nr][nc] === 0) {
            newBoard[nr][nc] = 2;
          }
          nr += dr;
          nc += dc;
        }
      }
    }
  }
  return newBoard;
}



What it does



  Simulates wind blowing in one compass direction. For every flower on the board, it
  traces outward step-by-step in that direction and fills any empty cells (0)
  with seeds (2). It passes through flowers and seeds without stopping — a
  key rule of the game.




The JS concepts



  const [dr, dc] = DIRECTIONS[direction] is destructuring assignment.
  It unpacks an array into named variables in one line — identical to Python's tuple unpacking
  dr, dc = DIRECTIONS[direction].




  The outer loops use let r and let c because those variables
  are reassigned on each iteration. The inner let nr, nc are also let
  because the while loop updates them. This is the rule of thumb: use
  let when you'll do x += something or x = newValue.




  The while loop bounds check nr >= 0 && nr < 5 && nc >= 0 && nc < 5
  is the JS equivalent of Python's 0 <= nr < 5 and 0 <= nc < 5. JS doesn't
  support chained comparisons, so you write each bound explicitly with &&.





  

    
JavaScript

    
const [dr, dc] = DIRECTIONS[dir];
// loop:
for (let r = 0; r < 5; r++) {
  for (let c = 0; c < 5; c++) { ... }
}
// bounds check:
nr >= 0 && nr < 5 && nc >= 0 && nc < 5

  

  

    
Python

    
dr, dc = DIRECTIONS[dir]
# loop:
for r in range(5):
  for c in range(5):
    ...
# bounds check:
0 <= nr < 5 and 0 <= nc < 5

5 checkWin




  Array.every()
  chained array methods
  !== operator




function checkWin(board) {
  return board.every(row => row.every(cell => cell !== 0));
}



What it does



  Returns true if no cell on the board is empty (no zeros remain).
  All cells must be either a flower (1) or a seed (2).
  One zero anywhere means the player loses.




The JS concepts



  array.every(fn) returns true only if the function returns
  true for every element. It short-circuits — as soon as one element fails,
  it stops checking. The Python equivalent is the built-in all().




  board.every(row => row.every(cell => cell !== 0)) chains two calls:
  the outer iterates rows, the inner iterates cells. The whole expression reads:
  "every row is such that every cell is not zero."





  

    
JavaScript

    
board.every(row =>
  row.every(cell => cell !== 0)
)
// also useful:
board.some(row =>
  row.some(cell => cell === 0)
) // true if ANY zero exists

  

  

    
Python

    
all(cell != 0
    for row in board
    for cell in row)

# any zero:
any(cell == 0
    for row in board
    for cell in row)

  






  every vs some: JS's .every() is Python's all(),
  and JS's .some() is Python's any(). Both short-circuit.

6 getValidPlacements




  Array.push()
  object literals
  || OR operator




function getValidPlacements(board) {
  const placements = [];
  for (let r = 0; r < 5; r++) {
    for (let c = 0; c < 5; c++) {
      if (board[r][c] === 0 || board[r][c] === 2) {
        placements.push({ row: r, col: c });
      }
    }
  }
  return placements;
}



What it does



  Returns a list of all cells where a flower can be placed — cells that are either empty
  (0) or seeded (2). Flowers (1) are the only
  excluded cells.




The JS concepts



  array.push(value) appends to an array in place, like Python's
  list.append(value). It returns the new length of the array, though
  that return value is rarely used.




  { row: r, col: c } is an object literal — an inline
  dictionary. This is idiomatic JS for returning structured data from a function. The
  Python equivalent would be a dict: {'row': r, 'col': c},
  or a namedtuple, or a dataclass instance.





  

    
JavaScript

    
placements.push({ row: r, col: c });
// access later:
placements[0].row
placements[0].col

  

  

    
Python

    
placements.append({'row': r, 'col': c})
# access later:
placements[0]['row']
placements[0]['col']

  






  A more JS-idiomatic alternative (not used here, but common) would be to build this
  with a flatMap: iterate rows and columns and return position objects for
  valid cells. But the imperative loop is easier to read and just as fast for a 5×5 board.

7 getUnusedDirections




  Object.keys()
  Array.filter()
  Array.includes()
  ! negation




function getUnusedDirections(usedDirections) {
  return Object.keys(DIRECTIONS).filter(d => !usedDirections.includes(d));
}



What it does



  Given a list of already-used direction strings, returns the ones not yet used.
  The game uses 7 of the 8 compass directions across 7 rounds, so this tells
  the wind which choices remain.




The JS concepts



  Object.keys(obj) returns an array of an object's own string keys — exactly
  like list(dict.keys()) in Python. It preserves insertion order (in practice),
  so for DIRECTIONS you get ['N', 'NE', 'E', 'SE', 'S', 'SW', 'W', 'NW'].




  array.filter(fn) returns a new array containing only elements for which
  the function returns true. The closest Python equivalent is a list
  comprehension with an if clause: [x for x in arr if condition(x)].
  JS also has Array.from(arr).filter(...) but plain .filter() is cleaner.




  array.includes(value) is like Python's value in list.
  !usedDirections.includes(d) reads "d is not in usedDirections."





  

    
JavaScript

    
Object.keys(DIRECTIONS)
  .filter(d => !usedDirections.includes(d))

// includes = Python's "in":
[1,2,3].includes(2)  // true

  

  

    
Python

    
[d for d in DIRECTIONS.keys()
 if d not in used_directions]

# 'in' test:
2 in [1,2,3]         # True

8 printBoard




  object as lookup table
  for...of loop
  Array.map() + join()
  template literals
  console.log




function printBoard(board) {
  const symbols = { 0: '.', 1: '*', 2: '\u00b7' };
  for (const row of board) {
    console.log(row.map(cell => symbols[cell]).join(' '));
  }
}



What it does



  Prints the board to the browser console in a human-readable grid.
  . = empty, * = flower, · = seed.
  Useful for debugging — open DevTools (F12) and you can inspect the board state at any point.




The JS concepts



  { 0: '.', 1: '*', 2: '·' } is a plain object used as a lookup table,
  mapping integers to display characters. Python would use a dict:
  {0: '.', 1: '*', 2: '·'}. Functionally identical — both support integer keys.




  for (const row of board) is the for...of loop — iterates over
  the values of an iterable, just like Python's for row in board. Use this
  when you don't need the index.




  row.map(cell => symbols[cell]).join(' ') chains two array methods.
  .map() transforms each cell value to its symbol character.
  .join(sep) concatenates the resulting array into a string with a separator —
  exactly like Python's ' '.join(list).




  console.log is JS's print. It accepts any number of arguments
  separated by commas and prints them space-separated. You can also use template literals
  for formatted output: console.log(`Round ${round} of 7`).





  

    
JavaScript

    
for (const row of board) {
  console.log(
    row.map(c => symbols[c]).join(' ')
  );
}
// template literal:
console.log(`Round ${round} of 7`);

  

  

    
Python

    
for row in board:
    print(
        ' '.join(symbols[c] for c in row)
    )
# f-string:
print(f'Round {round} of 7')

  






  Template literals use backticks (`), not quotes.
  ${expression} inside them evaluates and inserts the result — identical
  to Python's f-strings with {expression}. They also support multi-line
  strings without escape characters, which f-strings don't.
